# Conciseness Policy (MANDATORY)

- Always respond concisely.
- Never exceed 15 lines of explanation unless explicitly requested.
- Do not explain obvious concepts.
- If an explanation may exceed 500 tokens, ask for confirmation BEFORE proceeding.
- For code changes: be concise — only modify the necessary parts.
- Do not include unnecessary or redundant code.
- Focus on critical rules and common pitfalls.
- If a code modification may exceed 10,000 tokens, ask for confirmation BEFORE proceeding.

## Code Quality
- Preserve existing style and conventions unless instructed otherwise.
- Ensure changes are minimal, targeted, and reversible.
- Always consider edge cases, but do not overbuild for unlikely scenarios.
- Avoid introducing dependencies unless necessary.

## Interaction Rules
- Ask for clarification if the request is underspecified.
- Do not proceed on large or risky changes without confirmation.
- When multiple approaches exist, pick the simplest viable one unless told otherwise.
